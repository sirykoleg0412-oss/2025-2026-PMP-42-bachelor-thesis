# RBF-сурогатне моделювання екологічної системи «жертва – два хижаки»

Дипломна робота. Мова реалізації — Python 3.

## Постановка задачі

Розглядається тривидова екологічна система з функціональною відповіддю Холлінга II:

```
dN/dt  = r·N·(1 - N/K) - a₁·P₁·N/(1 + a₁·h₁·N) - a₂·P₂·N/(1 + a₂·h₂·N)
dP₁/dt = e₁·a₁·P₁·N/(1 + a₁·h₁·N) - d₁·P₁
dP₂/dt = e₂·a₂·P₂·N/(1 + a₂·h₂·N) - d₂·P₂
```

де N — чисельність жертви, P₁ та P₂ — два хижаки. Вектор параметрів **b** ∈ ℝ¹⁰:

| # | Параметр | Опис | Діапазон |
|---|---|---|---|
| 1 | r | швидкість росту жертви | [0.5, 3.0] |
| 2 | K | ємність середовища | [50, 300] |
| 3 | a₁ | частота атаки хижака 1 | [0.05, 0.40] |
| 4 | h₁ | час обробки хижака 1 | [0.05, 0.50] |
| 5 | e₁ | ефективність засвоєння хижака 1 | [0.20, 0.90] |
| 6 | d₁ | смертність хижака 1 | [0.10, 0.70] |
| 7 | a₂ | частота атаки хижака 2 | [0.05, 0.40] |
| 8 | h₂ | час обробки хижака 2 | [0.05, 0.50] |
| 9 | e₂ | ефективність засвоєння хижака 2 | [0.20, 0.90] |
| 10 | d₂ | смертність хижака 2 | [0.10, 0.70] |

Чисельний розв'язок ОДУ (солвер RK45, t ∈ [0, 50]) є **дорогою** операцією. Мета роботи — побудувати **швидкий сурогат** ψ̂(b), що апроксимує відгук системи:

```
ψ₁(b) = Σ N(tⱼ)²,   ψ₂(b) = Σ P₁(tⱼ)²,   ψ₃(b) = Σ P₂(tⱼ)²
```

## Метод

### Базова RBF-інтерполяція

Сурогат будується у вигляді лінійної комбінації радіальних базисних функцій:

```
ψ̂(b) = Σᵢ wᵢ · φ(‖b − bᵢ‖)
```

Підтримувані ядра φ:

| Ядро | Формула |
|---|---|
| Gaussian | exp(−ε·r²) |
| Multiquadric | √(1 + ε²r²) |
| Inverse Multiquadric | 1/√(1 + ε²r²) |

### Градієнтно-збагачений RBF (Hermite-інтерполяція)

До навчальної системи додаються рівняння на частинні похідні ∂ψ/∂bᵢ, обчислені числовим диференціюванням (центральні різниці). Це дозволяє отримати вищу точність при тій самій кількості навчальних точок.

### Адаптивне семплювання

Після базового навчання нові точки дорощуються з пулу кандидатів за двома стратегіями:
- **error** — точки з найбільшою похибкою поточного сурогата
- **gradient** — точки з найбільшою нормою ∇ψ (де функція найрізкіше змінюється)

## Сценарії

Задача розбита на 13 підсценаріїв залежно від кількості варійованих параметрів:

- **1D (6 сценаріїв):** r, K, a₁, d₁, a₂, d₂
- **2D (6 сценаріїв):** r\_K, a₁\_d₁, a₂\_d₂, d₁\_d₂, a₁\_a₂, e₁\_e₂
- **3D (1 сценарій):** r\_a₁\_a₂

Для кожного сценарію навчальна вибірка генерується методом **Latin Hypercube Sampling** з розмірами N ∈ {5, 10, 20, 40, 80}.

## Ключові результати

### R² залежно від N (найкраще ядро, multiquadric/IMQ)

| Сценарій | N=5 | N=10 | N=20 | N=40 | N=80 |
|---|---|---|---|---|---|
| K (1D) | 0.999 | 0.999 | 0.999 | 0.999 | 0.9997 |
| a₂ (1D) | 0.9993 | 0.9999 | 0.9999 | 0.9999 | 0.9999 |
| r\_K (2D) | 0.33 | 0.77 | 0.83 | 0.91 | 0.93 |
| a₁\_d₁ (2D) | 0.40 | 0.80 | 0.81 | 0.90 | **0.94** |
| a₂\_d₂ (2D) | −0.91 | 0.54 | 0.81 | 0.96 | **0.97** |
| e₁\_e₂ (2D) | 0.62 | 0.72 | 0.83 | 0.92 | **0.94** |
| r\_a₁\_a₂ (3D) | 0.20 | 0.22 | 0.63 | 0.78 | 0.58 |

Однопараметричні сценарії досягають R² > 0.999 навіть при N=5. Двопараметричні потребують N ≥ 40 для надійної апроксимації (R² > 0.90). Тривимірний сценарій залишається складнішим через «прокляття розмірності».

## Запуск

### 1. Встановлення залежностей

```bash
pip install -r requirements.txt
```

### 2. Перевірка моделі

```bash
python main.py              # режим ОДУ Холлінга II
python main.py rosenbrock   # тестовий полігон функції Розенброка
```

### 3. Генерація всіх результатів (по порядку)

```bash
# Спочатку — таблиці метрик (CSV), решта скриптів залежить від них
python scripts/generate_tables.py

# Порівняльні графіки + metrics_comparison.csv
python scripts/generate_comparison_plots.py

# 2D графіки (true vs predicted)
python scripts/generate_2d_plots.py

# Зведений графік RMSE vs N
python scripts/generate_summary_plot.py

# Верифікація на функції Розенброка
python scripts/rosenbrock_kernel_comparison.py
```

Результати зберігаються у `data/results/` (CSV) та `plots/` (PNG).
